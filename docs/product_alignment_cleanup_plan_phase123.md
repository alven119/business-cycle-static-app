# Phase 123：Live Ordered-Cycle Evidence 與轉折判讀 Runtime

Phase 123 沿用既有 book evidence rule registry、Phase 121 轉換公式與 boom-to-recession
lane contracts，將 NAS PostgreSQL 的 revised history 接入 declared boom 的即時研究判讀。
它不建立新的四階段分類器，也不修改 declared state。

## 完成內容

- 五個 transition-critical roles 以受治理的年增率或 4 期平均建立 evaluator input。
- 榮景延續、榮景結束觀察、衰退觀察與衰退確認四條 lane 顯示支持、反對、mixed、
  abstain、缺漏與 why-not-confirmation。
- 實質零售與廣義實質消費採 component consistency，不以平均或投票消除 mixed。
- 衰退確認要求就業與廣義消費兩個核心 confirmation inputs 同時支持；即使成立，
  仍不自動修改 declared 榮景。
- 首頁與服務 trust metadata 顯示 live evaluator 已接線及 revised diagnostic 邊界。

## 保留邊界

- raw value 與 smoothing 本身都不是 phase evidence。
- watch 不等於 confirmation；confirmation evidence 不等於 state-machine transition。
- 不輸出 candidate/current phase、phase score、phase rank、權重、交易或個人化配置。
- 本階段不執行 strict PIT replay、歷史績效回測或 prospective registry 寫入。
