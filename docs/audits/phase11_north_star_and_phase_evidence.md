# Phase 11 North Star and Phase Evidence

`specs/common/project_north_star_contract.yaml` as the highest-level product
contract for future work. Future phases must report North Star alignment,
advanced product capabilities, web surfaces, deferred gaps, semantic drift, and
production behavior changes.

The phase-evidence layer is shadow-only. It converts only safely
operationalizable book-core roles into governed research evidence. It does not
create candidate phases, current phases, state-machine input, resolver input,
dashboard production behavior, or portfolio actions.

Important separations preserved:

- Raw observation is not phase evidence.
- Watch is not confirmation.
- Candidate phase is not current phase.
- Revised diagnostic is not point-in-time result.
- Structural readiness is not economic validation.
- Portfolio research is not investment recommendation.

official equivalent or authorized source review. ADP employment requires
authorized reproducible access. Composite interpretation roles need complete
same-as-of and sustainable-rule semantics before they can emit phase support.
`recovery_publication_lag_awareness` is a temporal methodology requirement, not
an indicator denominator member.
