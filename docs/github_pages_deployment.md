# GitHub Pages Deployment

GitHub Pages now deploys the doctrine-aligned research dashboard. The Pages
artifact is generated by CI into ignored `public/` output and is not committed
to the main branch.

The deployed dashboard is the research surface for:

- declared cycle state and legal next transition context
- latest evidence drilldown
- indicator method explanations
- YTD / trailing 1Y / trailing 5Y chart payloads
- transition risk evidence accumulation
- portfolio policy research templates
- replay/backtest research surfaces and caveats

Legacy `cycle_snapshot` and `build_site.py` outputs remain available for
compatibility tests, but they are no longer the GitHub Pages deployment path.

## Enable GitHub Pages

In the GitHub repository:

1. Open `Settings`.
2. Open `Pages`.
3. Set Source to `GitHub Actions`.

The workflow uses the official Pages artifact flow:

- `actions/configure-pages`
- `actions/upload-pages-artifact`
- `actions/deploy-pages`

The deployed artifact path is `public`.

## Why public/ Is Not Committed

`public/` is generated output. It can contain derived dashboard JSON, rendered
HTML, local preview assets, and workflow-specific build output. The repository
keeps source code, specs, tests, and documentation under version control; CI
builds `public/` and uploads it as a Pages artifact.

## Workflow

The workflow lives at `.github/workflows/pages.yml`.

Every run first executes:

```bash
env -u FRED_API_KEY python -m pytest
ruff check .
```

The Pages build then runs:

```bash
python scripts/build_github_pages_research_dashboard.py --output-dir public
python scripts/validate_github_pages_research_dashboard.py --site-dir public
```

The validator checks that the Pages artifact includes:

- `public/index.html`
- `public/latest-evidence.html`
- `public/portfolio-replay.html`
- `public/data/dashboard_bundle.json`
- dashboard CSS and JS assets

It also checks that the rendered HTML contains the expected research dashboard
surfaces and no forbidden action fields.

## Manual Trigger

In the GitHub repository:

1. Open `Actions`.
2. Select `Build and deploy business cycle dashboard`.
3. Select `Run workflow`.

`workflow_dispatch` runs tests, lint, research dashboard build, validation,
Pages artifact upload, and deployment.

## Schedule

The schedule uses UTC:

```yaml
- cron: "17 10 * * 1"
```

Pushes to `main` or `master` run test and build sanity, but deployment is
restricted to manual or scheduled runs.

## Troubleshooting

`Pages source is not GitHub Actions`

Open `Settings` -> `Pages` and set Source to `GitHub Actions`.

`public/latest-evidence.html is missing`

Check the `Build doctrine-aligned research dashboard` step. The Pages workflow
must call `scripts/build_github_pages_research_dashboard.py`, not the legacy
`scripts/build_site.py`.

`Dashboard still looks like the old page`

Confirm the deployed workflow commit includes the Pages migration and hard
refresh the browser. The new artifact includes `latest-evidence.html` and
`portfolio-replay.html`; the old legacy homepage did not.

`Pages deployment failed, try again later`

GitHub Pages can occasionally fail after a valid artifact upload. Re-run the
workflow once. If it repeats, inspect the deploy job and repository Pages
settings.

## Safety

The Pages dashboard is a research dashboard. It does not provide personalized
allocation instructions, buy/sell signals, or trade actions. Portfolio policy
templates are research assumptions, not live recommendations.
